Limitations

Currently, the export package is created using Delivery API, and as such it cannot make an identical copy of your project. Some information is not available through Delivery API and is lost during export/import process. Once the Content managemenent API is mature enough, this application will switch to it to create better export packages with more information.

Current limitations include:

1) Content type snippets are not preserved, but data is not lost. Instead, these elements are created on each content type separately. 2) Components in rich text elements are created as standalone linked items (they are preserved correctly in rich text values) 3) Content type configuration is lost (e.g. size limitations for assets, allowed content types for linked item elements...) 4) Only published items are exported 5) Assets folder categorization is not preserved and assets are created in flat structure
